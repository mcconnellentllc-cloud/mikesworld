Built with Claude Code

How AI-assisted development brought this website to life — and what comes next.

What is Claude Code?

Claude Code is an AI-powered command-line tool built by Anthropic. It acts as a coding partner that can read, write, and edit code directly in your project through natural conversation. Instead of manually writing every line of HTML, CSS, and JavaScript, we described what we wanted and Claude Code built it — iterating with us in real time.

Think of it like having a senior developer sitting next to you. You say, "Build a products page with our beef offerings, a gold and black color scheme, and mobile responsiveness," and it creates the files, writes the code, and refines the design based on your feedback.

How This Site Was Built

This entire website — every page, every style, every interaction — was built through a conversational workflow with Claude Code. Here's how the process worked:

  1. Started with the vision. We described the Pasture Perfect brand: a family farm in Sedgwick, Colorado, raising premium Black Angus beef. We wanted a premium, elegant feel — black and gold, refined typography, no clutter.
  2. Built the foundation. Claude Code generated the HTML structure, CSS styling, and JavaScript functionality for the homepage, including navigation, hero section, testimonials, and payment options.
  3. Expanded page by page. Through conversation, we added the About page, Products page, Our History timeline, FAQ accordion, Contact form, and Cut Sheet — each matching the established design language.
  4. Refined the details. We iterated on colors, fonts (Cormorant Garamond for headings, Inter for body text), spacing, and mobile responsiveness. Adjustments were made in real time through simple requests.
  5. Managed with Git. Claude Code handled version control throughout — creating branches, committing changes, and pushing to GitHub, all from the same conversation.

What We Learned

Building with Claude Code fundamentally changes the development process. Here are some key takeaways:

  • Speed. What would typically take days or weeks of development was accomplished in hours. Full pages with responsive design, animations, and interactive elements were created through simple descriptions.
  • Iteration is effortless. Changing the entire color scheme from warm browns to black and gold? One request. Adding six testimonials instead of three? Done in seconds. Removing emojis across every page? Handled instantly.
  • No coding knowledge required to direct. While understanding code helps, the conversational interface means anyone with a clear vision can guide the development process.
  • Quality output. The generated code follows best practices — semantic HTML, CSS custom properties, accessible markup, mobile-first responsive design.

Next Steps: Adding a Database with Airtable

Currently, the Cut Sheet and Contact forms use mailto: links to send information via email. This works, but it means data isn't stored in a central location. The next phase is to connect the site to Airtable as a lightweight database backend.

Why Airtable?

For a small family operation with low traffic, Airtable is an ideal choice:

  • Always on. Unlike many free database services (Render, Supabase, Neon) that spin down or pause after periods of inactivity, Airtable is always available. No cold starts, no waiting for the database to wake up.
  • Visual interface. Kim can view, sort, filter, and manage all orders and cut sheets directly in Airtable's spreadsheet-like interface — no technical knowledge needed.
  • Free tier works for low volume. The free plan includes 1,000 records per base and 1,000 API calls per month. For a small beef operation processing a handful of orders per week, this is more than enough.
  • Built-in notifications. Airtable can be configured to send email notifications when new records are added, so Kim gets alerted immediately when a customer submits a cut sheet.
  • No server required. The website can send data directly to Airtable's API from the browser, eliminating the need to set up and maintain a backend server.

Implementation Roadmap

Here's how we'll connect the site to Airtable:

01

Create the Airtable Base

Set up an Airtable base with two tables: Cut Sheet Submissions and Contact Form Messages. Each table will have fields matching our form inputs (customer name, email, phone, order type, cut preferences, etc.).

02

Generate an API Key

Create a personal access token in Airtable with write permissions to the base. This token will be used to authenticate API requests from the website forms.

03

Update the Form Handlers

Modify the JavaScript form handlers to send data to Airtable's REST API instead of (or in addition to) the mailto: link. Each form submission will create a new record in the appropriate table.

04

Add Confirmation & Error Handling

Show the customer a success message when their submission is recorded, or a helpful error message if something goes wrong. Include a fallback to the email method if the API is unavailable.

05

Set Up Notifications

Configure Airtable automations to email kim@theperfectbeef.com whenever a new cut sheet or contact form submission arrives. This ensures no order is missed.

06

Optional: Add Order Tracking

Add status fields in Airtable (Received, Processing, Ready for Pickup, Shipped) so Kim can track each order's progress. A future enhancement could let customers check their order status on the website.

Example: How the API Call Works

Here's a simplified example of how a cut sheet submission would be sent to Airtable:

// Send cut sheet data to Airtable
fetch('https://api.airtable.com/v0/YOUR_BASE_ID/Cut%20Sheets', {
  method: 'POST',
  headers: {
    'Authorization': 'Bearer YOUR_API_TOKEN',
    'Content-Type': 'application/json'
  },
  body: JSON.stringify({
    fields: {
      'Name': 'John Smith',
      'Email': 'john@example.com',
      'Order Type': 'Half Beef',
      'Steak Thickness': '1 inch',
      'Roast Size': '3-4 lbs',
      'Ground Beef Pkg': '1 lb packages'
      // ... all other cut sheet fields
    }
  })
});

Security Note

For production use, the Airtable API token should not be exposed in client-side JavaScript. A lightweight serverless function (via Vercel, Netlify, or Cloudflare Workers — all with free tiers) can act as a proxy to keep the token secure while still allowing the static site to submit data.
